R projects come in two flavors: analysis projects (RStudio Projects) and packages. Analysis projects follow a loose convention; packages have a strict structure enforced by `R CMD check`. The `usethis` package scaffolds both.

## Example

```
# R Package structure
my_package/
├── R/                       # R source files
│   ├── utils.R
│   └── models.R
├── man/                     # generated documentation (.Rd files)
├── tests/
│   └── testthat/
│       ├── helper.R
│       └── test-utils.R
├── vignettes/               # long-form documentation
├── data/                    # .rda datasets
├── inst/                    # arbitrary installed files
├── DESCRIPTION              # package metadata
├── NAMESPACE                # exported symbols
└── README.md

# Analysis project (RStudio Project)
my_analysis/
├── data/
│   ├── raw/
│   └── processed/
├── R/
│   ├── 01_load_data.R
│   ├── 02_clean.R
│   └── 03_model.R
├── reports/
│   └── analysis.Rmd
├── renv/
├── renv.lock
└── my_analysis.Rproj
```

## Gotchas

- `DESCRIPTION` and `NAMESPACE` are mandatory for R packages; `NAMESPACE` is usually auto-generated by `roxygen2`
- Never use `setwd()` in scripts that others will run — use `here::here()` for portable paths
- `devtools::load_all()` sources all files in `R/` for interactive development
