Mix projects follow a standard layout generated by `mix new`. The application code lives under `lib/`, tests under `test/`. OTP applications (with `--sup`) add an `Application` module and a supervision tree.

## Example

```
my_app/
├── mix.exs              -- project manifest & dependencies
├── mix.lock             -- pinned dependency versions
├── lib/
│   ├── my_app.ex        -- Application module (if --sup)
│   └── my_app/
│       ├── repo.ex      -- MyApp.Repo
│       ├── user.ex      -- MyApp.User
│       └── user_service.ex  -- MyApp.UserService
├── test/
│   ├── test_helper.exs  -- ExUnit.start()
│   └── my_app/
│       ├── user_test.exs
│       └── user_service_test.exs
├── config/
│   ├── config.exs       -- shared config
│   ├── dev.exs          -- development config
│   ├── test.exs         -- test config
│   └── prod.exs         -- production config (or runtime.exs)
└── priv/                -- migrations, seeds, static assets
```

```elixir
# lib/my_app.ex (OTP Application)
defmodule MyApp do
  use Application

  def start(_type, _args) do
    children = [
      MyApp.Repo,
      {MyApp.Worker, []}
    ]
    Supervisor.start_link(children, strategy: :one_for_one)
  end
end
```

```elixir
# lib/my_app/user.ex
defmodule MyApp.User do
  defstruct [:id, :name, :email]
end
```

## Gotchas

- Module names must match file paths: `MyApp.UserService` should be in `lib/my_app/user_service.ex`
- `config/runtime.exs` (Elixir 1.11+) is evaluated at runtime (not compile time) — use it for environment variables in production
- Keep `test/test_helper.exs` minimal; put shared test fixtures in `test/support/`
